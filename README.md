# Crypto Dashboard (Vue 3 + TS)

Небольшое SPA на Vue 3 для просмотра актуальных курсов криптовалют и их динамики.

## Стек

- Vue 3 + Composition API + TypeScript
- Pinia
- Vue Router
- Vite
- Chart.js + vue-chartjs
- Архитектура в стиле FSD: `app / pages / widgets / features / entities / shared`
- Coincap v3 API c авторизацией по Bearer-токену

## Настройка окружения

1. Надо скопировать из примера в енв файл, я там оставил апи кей, чтобы у вас сработало `.env.example` в `.env.local`:

```bash
cp .env.example .env.local
```

2. В `.env.local` Я оставил реальные ключи


## Запуск

```bash
npm install
npm run dev
```

## Архитектура

1. ### src/app — инфраструктура приложения

Инициализация приложения (main.ts)

Глобальные плагины

Маршрутизация (Vue Router)

Глобальные стили и темы

Конфигурация API-клиента

2. ### src/shared — переиспользуемые сущности

Здесь лежат:

UI-компоненты

утилиты и форматтеры

работа с локальным хранилищем (избранное)

общие типы данных (криптовалюты, графики и т.д.)

Это дает нам возможность не дублироваться

3. ### src/entities — бизнес-логика сущностей

Каждая сущность (Coin, Market, Chart) имеет:

модель данных

API-методы

хелперы

адаптеры для приведения данных ко внутреннему формату

Это дает нам возможность как бы намного легче переиспользовать бизнес-логику на разных страницах


4. ### src/features — функциональные модули

Здесь реализованы независимые фичи:

избранное

поиск

переключение темы

таблица криптовалют

карточка актива

графики

Тут фичи изолированы и могут использоваться на другиж страницах


5. ### src/pages — страницы приложения

/ — список криптовалют

/coin/:id — страница конкретного актива

Каждая страница состоит только из композиции готовых фич и сущностей — логики внутри страниц нет.

Данные взял из Coincap API v3 (`/assets`, `/assets/{slug}`, `/assets/{slug}/history`), все запросы идут с Bearer-токеном из `VITE_API_KEY`.
